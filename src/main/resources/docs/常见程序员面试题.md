# 常见程序员面试题 (含解析)

## 一、Java 核心基础

### 1. HashMap 的底层原理是什么？
- **数据结构**：JDK 1.8 之前是 **数组 + 链表**，JDK 1.8 之后是 **数组 + 链表 + 红黑树**。
- **红黑树转化**：当链表长度大于 8 且数组长度大于 64 时，链表会转为红黑树，以提高查询效率 (O(n) -> O(logn))。
- **扩容机制**：默认初始容量为 16，负载因子为 0.75。当元素个数超过 `容量 * 负载因子` 时，容量翻倍，并重新计算每个元素的位置 (Rehash)。
- **线程安全**：HashMap 是线程不安全的，多线程环境下建议使用 `ConcurrentHashMap`。

### 2. String, StringBuilder, StringBuffer 的区别？
- **String**：不可变字符序列，底层是 `final char[]` (Java 9+ 是 `byte[]`)。每次修改都会生成新对象，效率低，但线程安全。
- **StringBuilder**：可变字符序列，效率高，但**线程不安全**。适用于单线程环境下的大量字符串拼接。
- **StringBuffer**：可变字符序列，方法都加了 `synchronized`，**线程安全**，但效率比 StringBuilder 低。

### 3. == 和 equals() 的区别？
- **==**：
    - 基本数据类型：比较的是**值**是否相等。
    - 引用数据类型：比较的是**内存地址**是否相等。
- **equals()**：
    - 默认情况下比较内存地址 (Object 类中的实现)。
    - 通常会被重写 (如 String, Integer)，重写后比较的是**内容**是否相等。
    - **注意**：重写 `equals` 必须重写 `hashCode`。

## 二、并发编程 (JUC)

### 1. 线程池的核心参数有哪些？
1. `corePoolSize`：核心线程数。
2. `maximumPoolSize`：最大线程数。
3. `keepAliveTime`：多余空闲线程存活时间。
4. `unit`：时间单位。
5. `workQueue`：任务队列 (如 LinkedBlockingQueue)。
6. `threadFactory`：线程工厂。
7. `handler`：拒绝策略 (AbortPolicy, CallerRunsPolicy, DiscardOldestPolicy, DiscardPolicy)。

### 2. synchronized 关键字的底层原理？
- **实现机制**：基于 JVM 的 `Monitor` 对象实现。代码块使用 `monitorenter` 和 `monitorexit` 指令。
- **锁升级**：无锁 -> 偏向锁 -> 轻量级锁 -> 重量级锁。目的是为了减少重量级锁带来的性能消耗。
- **原子性/可见性**：保证了操作的原子性、可见性和有序性。

## 三、Spring 框架

### 1. Spring 的 IoC 和 AOP 理解？
- **IoC (控制反转)**：将对象的创建、初始化、销毁等生命周期交给 Spring 容器管理。开发者通过 DI (依赖注入) 获取对象，降低了耦合。
- **AOP (面向切面编程)**：在不修改源码的情况下，通过动态代理 (JDK 或 CGLIB) 对业务逻辑进行增强。常见场景：日志记录、事务管理、权限校验。

### 2. Spring Bean 的生命周期？
1. **实例化** (Instantiation)：反射创建对象。
2. **属性赋值** (Populate)：填充依赖。
3. **初始化** (Initialization)：
    - 执行 Aware 接口方法 (如 `setBeanName`)。
    - 执行 `BeanPostProcessor` 的前置处理。
    - 执行 `@PostConstruct` 或 `init-method`。
    - 执行 `BeanPostProcessor` 的后置处理。
4. **销毁** (Destruction)：执行 `@PreDestroy` 或 `destroy-method`。

## 四、数据库 (MySQL)

### 1. 为什么 MySQL 使用 B+ 树作为索引结构？
- **IO 效率高**：非叶子节点只存键值信息，不存数据，使得单节点能容纳更多键值，树的高度更低 (通常 3-4 层即可支撑千万级数据)。
- **范围查询强**：叶子节点之间有双向链表连接，非常适合 `>`、`<`、`BETWEEN` 等范围查询。
- **性能稳定**：所有查询最终都要到叶子节点，查询路径长度一致。

### 2. 事务的 ACID 特性？
- **原子性 (Atomicity)**：事务是最小单位，要么全部成功，要么全部回滚。
- **一致性 (Consistency)**：事务执行前后，数据库的完整性约束没有被破坏。
- **隔离性 (Isolation)**：并发事务之间互不干扰。隔离级别有：读未提交、读已提交、可重复读 (MySQL 默认)、串行化。
- **持久性 (Durability)**：事务提交后，修改是永久性的。

## 五、算法与逻辑

### 1. 快速排序的思想？
- **分治法**：选取一个基准元素 (Pivot)，将数组分为两部分，左边比基准小，右边比基准大。
- **递归**：对左右两部分递归进行同样的操作。
- **复杂度**：平均时间复杂度 O(nlogn)，最坏 O(n²)。

### 2. 什么是幻读？MySQL 是如何解决的？
- **幻读**：在一个事务内，两次相同的查询结果条数不一致 (因为其他事务插入了新数据)。
- **解决**：MySQL InnoDB 引擎在 **可重复读 (RR)** 级别下，通过 **MVCC (多版本并发控制)** 解决普通查询的幻读，通过 **Next-Key Lock (临键锁)** 解决当前读 (加锁查询) 的幻读。
