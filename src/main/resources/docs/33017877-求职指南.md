# 33017877 - 求职指南 (进阶实战版)

## 一、简历优化：如何写出让 HR 眼前一亮的简历？

### 1. 技能栈的艺术
- **分级描述**：不要全篇都是“精通”，建议使用：
    - **精通/深入理解**：Java 并发编程、JVM 调优、Spring 核心源码。
    - **熟练掌握**：Redis、MySQL 索引优化、RabbitMQ。
    - **熟悉/了解**：Docker、Kubernetes、Vue.js。
- **排版建议**：核心技能放在最前面，项目经历占 60% 以上篇幅。

### 2. STAR 原则实战示例
- **[反面教材]**：参与了电商系统的秒杀模块开发，解决了高并发问题。
- **[正面教材]**：
    - **S (背景)**：在双十一期间，秒杀模块预计面临 10W+ QPS 的瞬时流量。
    - **T (任务)**：负责秒杀核心接口的性能优化，确保系统不崩溃且数据一致。
    - **A (行动)**：
        1. 使用 **Redis 预热**库存信息，配合 **Lua 脚本**实现原子扣减，减少 DB 压力。
        2. 引入 **RocketMQ** 进行流量削峰，异步处理订单入库。
        3. 开启 **Sentinel** 热点参数限流，防止恶意刷单。
    - **R (结果)**：接口响应时间从 500ms 降至 50ms，成功支撑了 15W QPS 的峰值，且未发生超卖现象。

## 二、面试中的“潜规则”与技巧

### 1. 自我介绍的模板 (3分钟)
- **第1分钟**：基本情况、教育背景、职业目标。
- **第2分钟**：核心竞争力 (你最擅长的 2-3 个技术点)。
- **第3分钟**：最近一个项目的亮点及你的贡献。

### 2. 如何回答“你有什么缺点？”
- **避坑**：不要说“我太追求完美” (太假)，也不要说“我脾气不好” (致命)。
- **话术**：说一个与工作核心能力无关，且你正在改进的缺点。
    - *示例*：“我早期对前端框架了解较浅，导致在全栈沟通时效率不够高。最近半年我正在学习 Vue.js，目前已经能独立完成简单的管理后台开发。”

### 3. 遇到不会的问题怎么办？
- **第一步**：坦诚告知对此点了解不深。
- **第二步**：展示你的逻辑思维。
    - *话术*：“虽然我没有直接使用过这个组件，但根据我的理解，它的原理可能类似于 XXX，我会通过 XXX 的方式去解决这类问题……”

## 三、面试后的黄金复盘

### 1. 建立“面试错题本”
- 记录面试官追问的深度。比如他问了 HashMap，追问到红黑树左旋右旋了吗？
- 记录自己表达不顺畅的地方，录音复听是一个极好的习惯。

### 2. 感谢信/面试反馈
- 面试结束后 24 小时内，可以通过 HR 传达对面试官专业度的认可及对职位的兴趣。

## 四、常见求职 Q&A

- **Q: 零经验/转行如何找工作？**
    - A: 突出你的自学能力和开源贡献。建立一个高质量的 GitHub 仓库，并在简历中附上链接。
- **Q: 被压薪资怎么办？**
    - A: 了解该城市的行业平均水平，强调你的入职即战力。如果薪资不理想，可以尝试谈更多的福利或期权。
- **Q: 如何平衡多份 Offer？**
    - A: 优先考虑：**技术成长空间 > 团队氛围 > 薪资待遇 > 离家远近**。
